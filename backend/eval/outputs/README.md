# Evaluation Outputs

This directory stores all evaluation results, error logs, and analysis reports.

## File Types

### Result Files
- `eval-results-YYYY-MM-DD.json`: Complete detailed evaluation results
- `eval-summary-YYYY-MM-DD.md`: Human-readable summary report

### Error Files  
- `eval-errors-YYYY-MM-DD.jsonl`: Failed test cases with categorized failures

### Trace Files (Optional)
- `eval-trace-YYYY-MM-DD.jsonl`: Debug traces when TRACE_EXPORT=true

### Analysis Files
- `eval-trends.json`: Historical trend data and analysis
- `eval-trends-summary.md`: Trend analysis in readable format
- `fragile-cases-report.md`: Cases that fail frequently

## Privacy & Safety

**IMPORTANT**: These output files are designed to be safe for sharing:

✅ **Safe to Share:**
- All JSON result files (no PII)
- Error logs (PII redacted)
- Analysis reports
- Trend data

❌ **Never Stored:**
- Raw transcript text (except in dataset files)
- Real names/emails/phones
- User profile IDs
- Donor information

## Retention Policy

- Keep result files for 90 days minimum
- Archive old files after 6 months
- Always retain trend analysis files
- Clean up trace files after 30 days (large size)

## File Structure Example

```
outputs/
├── eval-results-2026-01-15.json     # Detailed results
├── eval-summary-2026-01-15.md       # Summary report  
├── eval-errors-2026-01-15.jsonl     # Error records
├── eval-trace-2026-01-15.jsonl      # Debug traces (if enabled)
├── eval-trends.json                 # Historical trends
├── eval-trends-summary.md           # Trend report
└── fragile-cases-report.md          # Frequently failing cases
```

## Usage

Files in this directory are automatically generated by:

```bash
npm run eval:parsing          # Creates result/error/summary files
npm run eval:parsing:trace    # Also creates trace files  
npm run eval:analyze         # Creates trend/fragile case reports
```
